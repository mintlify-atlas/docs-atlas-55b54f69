## Overview

The Bounce Project uses **Facebook Prophet**, a time-series forecasting model, to predict stock prices up to 365 days into the future. By analyzing historical price data from Yahoo Finance, the system generates data-driven forecasts with confidence intervals.

## Data Collection

The system fetches stock data using the yfinance library:

```python
# From app.py:26-30
import yfinance as yf

def fetch_stock_data(ticker_symbol, start_date, end_date):
    stock_data = yf.download(ticker_symbol, start=start_date, end=end_date)
    df = stock_data[['Adj Close']].reset_index()
    df = df.rename(columns={'Date': 'ds', 'Adj Close': 'y'})
    return df
```

### Data Format

Prophet requires a specific DataFrame structure:

| Column | Description | Example |
|--------|-------------|--------|
| `ds` | Date (datetime) | 2024-01-15 |
| `y` | Target value (adjusted close price) | 185.42 |

<Note>
  **Adjusted Close Price** accounts for stock splits, dividends, and other corporate actions, providing more accurate historical values than raw closing prices.
</Note>

## Model Training

Training Prophet is straightforward:

```python
# From app.py:33-36
from prophet import Prophet

def train_prophet_model(df):
    model = Prophet()
    model.fit(df)
    return model
```

Prophet automatically:
- Detects trend changes (changepoints)
- Identifies seasonality (weekly, yearly)
- Handles outliers and missing values
- Fits the model using Stan optimization

<Tip>
  Prophet uses **Bayesian inference** under the hood, which allows it to provide uncertainty estimates (confidence intervals) for predictions.
</Tip>

## Generating Forecasts

The system predicts 365 days into the future:

```python
# From app.py:40-43
def make_forecast(model, periods):
    future = model.make_future_dataframe(periods=periods)
    forecast = model.predict(future)
    return forecast
```

### Forecast Output

The forecast DataFrame contains:

| Column | Description |
|--------|-------------|
| `ds` | Future dates |
| `yhat` | Predicted value (point estimate) |
| `yhat_lower` | Lower bound of 95% confidence interval |
| `yhat_upper` | Upper bound of 95% confidence interval |
| `trend` | Overall trend component |
| `seasonal` | Seasonal component |

import Plotly from "plotly.js-dist-min";

const StockHistoryGraph = ({ dataframe }) => {
  useEffect(() => {
    if (dataframe !== null && dataframe.data) {
      // Extract dates (column 0) and predictions (column 18 = yhat)

        y: dy,

};

## Interpreting Predictions

### Understanding the Forecast Chart

<Steps>
  <Step title="Historical Data (Blue/White)">
    Past stock prices show actual performance and establish the baseline trend
  </Step>
  
  <Step title="Predicted Prices (Forecast Line)">
    The continuation shows Prophet's point estimates for future prices
  </Step>
  
  <Step title="Confidence Bands (Shaded Area)">
    The wider the band, the more uncertainty in the prediction. Near-term predictions are typically more confident.
  </Step>
  
  <Step title="Trend Direction">
    Upward slope = bullish forecast, downward slope = bearish forecast, flat = stable
  </Step>
</Steps>

### Example Interpretation

Stock: AAPL (Apple)

Interpretation:

<Warning>
  **Important Limitations:**
  - Prophet assumes past patterns will continue (not always true for stocks)
  - Cannot predict unexpected events (earnings surprises, regulatory changes, etc.)
  - Confidence intervals widen significantly for long-term predictions
  - Not financial advice - use as one of many analysis tools
</Warning>

## Model Configuration

Prophet can be customized with various parameters (currently using defaults):

```python
# Default configuration (current implementation)
model = Prophet()

# Advanced configuration (optional enhancements)
model = Prophet(
    daily_seasonality=False,      # Stocks don't have daily patterns
    weekly_seasonality=True,      # Market open/close patterns
    yearly_seasonality=True,      # Annual cycles
    changepoint_prior_scale=0.05, # Flexibility of trend changes
    seasonality_prior_scale=10,   # Strength of seasonality
)
```

<Tip>
  For production use, consider tuning hyperparameters using cross-validation on historical data to optimize prediction accuracy.

## Advanced: Prophet Decomposition

Prophet breaks predictions into components:

```
Prediction = Trend + Seasonality + Holidays + Error

For AAPL on 2025-06-15:
  yhat (198.45) = trend (195.10) + seasonal (3.35) + 0 + 0
```

This decomposition helps understand:
- **Trend**: Long-term direction (up/down)
- **Seasonality**: Recurring patterns (e.g., January effect)
- **Holidays**: Market closures (NYSE holidays)

<Note>
  The current implementation doesn't explicitly model holidays. Consider adding US market holidays for improved accuracy:
  ```python
  from prophet import Prophet
  model = Prophet(holidays=us_market_holidays)
  ```
</Note>
