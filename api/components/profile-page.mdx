## Overview

The `ProfilePage` component displays user profile information and their stock portfolio. It shows the user's profile picture, name, and a list of their stock investments.

## Component Structure

Located at `client/src/Components/ProfilePage.jsx`, this component provides:

- User profile section with avatar and name
- Portfolio overview with stock holdings
- Static portfolio display (currently with placeholder data)

## Props

This component does not accept any props. The current implementation uses hardcoded data.

<Note>
  The ProfilePage currently displays static data ("Sumit singh" and "Bank Nifty" holdings). In a production implementation, this should be replaced with data fetched from the `/showstocklist/` API endpoint.
</Note>

## Styling

The component uses the following design system:

- **Card Background**: Black with shadow (`bg-black shadow-xl`)
- **Avatar**: Circular image (32x32 rem) using Gravatar
- **Typography**: Monospace font (`font-mono`) for clean display
- **Hover Effects**: Blue hover state on stock items (`hover:text-blue-500`)
- **Layout**: Flexbox for responsive alignment

## Current Limitations

<Warning>
  The ProfilePage has several limitations in the current implementation:
  
  - **Static Data**: User name and portfolio holdings are hardcoded
  - **No API Integration**: Does not fetch real data from the `/showstocklist/` endpoint
  - **Missing Features**: No stock price information, performance indicators, or transaction history
  - **Placeholder Avatar**: Uses a generic Gravatar image
</Warning>

## Integration with Backend API

To make this component functional, integrate with the backend stock operations API:

```javascript Fetching Portfolio Data
const fetchPortfolio = async (username, token) => {
  const response = await fetch(
    `/showstocklist/?token=${encodeURIComponent(token)}`,
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ username })
    }
  );
  
  const data = await response.json();
  return data;
};
```

## Database Schema Reference

The profile page should display data from the `stocks` table:

| Column | Type | Description |
|--------|------|-------------|
| `ticket` | varchar(20) | Stock ticker symbol |
| `username` | varchar(20) | User who owns the stock |
| `amount` | integer | Investment amount |
| `dateOfInvestment` | timestamp | Purchase date |
