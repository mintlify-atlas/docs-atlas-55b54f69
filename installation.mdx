## System Requirements

Before installing The Bounce Project, ensure your system meets these requirements:

<CardGroup cols={2}>
  <Card title="Node.js" icon="node-js">

  <Card title="Python" icon="python">
    Version 3.12 or higher
  </Card>
  
  <Card title="Package Manager" icon="box">
    pnpm (recommended) or npm

  <Card title="Memory" icon="memory">

</Note>

## Installation Steps

### 1. Clone the Repository

Clone The Bounce Project from GitHub:

```bash
git clone https://github.com/vineetprash/byteClub_AB05.git
cd byteClub_AB05
```

The repository structure looks like this:

```
byteClub_AB05/
├── client/          # React frontend
├── server/          # Python backend
├── flagged/         # Gradio flagged data
└── README.md
```

### 2. Client Installation

The client is a React application built with Vite that provides the user interface.

<Steps>
  <Step title="Navigate to Client Directory">
    ```bash
    cd client
    ```
  </Step>
  
  <Step title="Install Dependencies">
    Using pnpm (recommended):
    ```bash
    pnpm install
    ```
    
    Or using npm:
    ```bash
    npm install
    ```
  </Step>
  
  <Step title="Verify Installation">
    Check that all dependencies were installed successfully:
    ```bash
    pnpm list
    ```
  </Step>
</Steps>

### 3. Server Installation

The server is a Python application that handles AI model inference, data fetching, and API endpoints.

<Steps>
  <Step title="Navigate to Server Directory">
    ```bash
    cd ../server
    ```
  </Step>
  
  <Step title="Install with pip">
    Install the server package and all dependencies:
    ```bash
    pip install .
    ```
    
    <Warning>
      This step will download large machine learning models including PyTorch and Transformers. It may take 10-20 minutes depending on your internet connection.
    </Warning>
  </Step>
  
  <Step title="Verify Installation">
    Check that key packages are installed:
    ```bash
    pip list | grep -E "gradio|transformers|prophet|yfinance"
    ```
  </Step>
</Steps>

## Configuration

### Backend Configuration

The backend server (`server/server/app.py`) includes several configurable parameters:

#### API Token Configuration

The system uses EdenAI for natural language generation. Update the token in `app.py:13`:

```python server/server/app.py
headers = {
    "Authorization": "Bearer YOUR_EDENAI_TOKEN_HERE"
}
```

<Tip>
  Get your EdenAI token at [https://www.edenai.co/](https://www.edenai.co/). The free tier should be sufficient for testing.
</Tip>

#### Stock Data Parameters

Modify the date range for historical stock data in `app.py:118-121`:

```python server/server/app.py
def make_graph(ticker):
    ticker_symbol = ticker
    start_date = "2020-01-01"  # Modify start date
    end_date = "2024-03-01"    # Modify end date
```

#### Forecast Period

Change the prediction window in `app.py:124`:

```python server/server/app.py
forecast = make_forecast(model, 365)  # Days to forecast (default: 365)
```

#### Sentiment Model

The sentiment analysis model is configured in `app.py:17`:

```python server/server/app.py
MODEL = "cardiffnlp/twitter-roberta-base-sentiment-latest"
tokenizer = AutoTokenizer.from_pretrained(MODEL)
model = AutoModelForSequenceClassification.from_pretrained(MODEL)
```

### Frontend Configuration

The frontend connects to the Gradio backend at a configurable URL.

#### Backend URL

If running the backend on a different port or host, update `client/src/App.jsx:20`:

```jsx client/src/App.jsx
const app = await client("http://127.0.0.1:7860/");
```

For production deployments, change this to your production backend URL:

```jsx client/src/App.jsx
const app = await client("https://your-backend-url.com/");
```

## Running the Application

### Development Mode

<Steps>
  <Step title="Start Backend Server">
    In one terminal:
    ```bash
    cd server/server
    python app.py
    ```
    
    Wait for the Gradio server to start:
    ```
    Running on local URL:  http://127.0.0.1:7860
    ```
  </Step>
  
  <Step title="Start Frontend">
    In another terminal:
    ```bash
    cd client
    pnpm run dev
    ```
    
    The Vite dev server will start:
    ```
    ➜  Local:   http://localhost:5173/
    ```
  </Step>
  
  <Step title="Access Application">
    Open your browser to `http://localhost:5173`
  </Step>
</Steps>

### Production Build

#### Build Frontend

```bash
cd client
pnpm run build
```

This creates an optimized production build in `client/dist/`.

#### Preview Production Build

```bash
pnpm run preview
```

<Info>
  For production deployment, serve the `dist/` folder using a static file server like Nginx or deploy to platforms like Vercel or Netlify.
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="PyTorch Installation Issues">
    If PyTorch fails to install, try installing it separately first:
    
    ```bash
    # For CPU-only (smaller download)
    pip install torch --index-url https://download.pytorch.org/whl/cpu
    
    # Then install the server package
    pip install .
    ```
    
    For GPU support, visit [PyTorch's official installation guide](https://pytorch.org/get-started/locally/).
  </Accordion>
  
  <Accordion title="Prophet Installation Errors">
    Prophet requires additional system dependencies. On Ubuntu/Debian:
    
    ```bash
    sudo apt-get install python3-dev
    sudo apt-get install build-essential
    ```
    
    On macOS:
    ```bash
    brew install gcc
    ```
  </Accordion>
  
  <Accordion title="pnpm Not Found">
    Install pnpm globally:
    
    ```bash
    npm install -g pnpm
    ```
    
    Or use npm instead throughout the installation.
  </Accordion>
  
  <Accordion title="Port Already in Use">
    If port 7860 (backend) or 5173 (frontend) is already in use:
    
    **Backend**: Gradio will automatically try the next available port. Check the console output for the actual URL.
    
    **Frontend**: Specify a different port:
    ```bash
    pnpm run dev -- --port 3000
    ```
  </Accordion>
  
  <Accordion title="CORS Issues">
    If you encounter CORS errors when the frontend connects to the backend, ensure both servers are running and the backend URL in `App.jsx` matches the actual Gradio server address.
  </Accordion>
  
  <Accordion title="Model Download Timeouts">
    If Hugging Face model downloads timeout:
    
    ```bash
    # Increase timeout
    export HF_HUB_DOWNLOAD_TIMEOUT=300
    
    # Then retry installation
    pip install .
    ```
  </Accordion>
</AccordionGroup>
